Button System Reference

Standardized button components for GBS EMEA Learning Hub

Three-Tier Button Hierarchy

Our button system follows a clear visual hierarchy to guide users through different action priorities:

Tier 1

Primary Buttons

Gradient buttons for main actions and CTAs

Tier 2

Secondary Buttons

Outline buttons for alternative actions

Tier 3

Tertiary Buttons

Text links for navigation and low-priority actions

Tier 1

Primary Buttons

Usage: Main call-to-action buttons, important actions, form submissions

Default (Blue)

Usage Guidelines

‚úÖ Do

  • Use primary buttons for the most important action on a page
  • Limit to 1-2 primary buttons per section
  • Match button colors to section themes (green for beginner, purple for advanced)
  • Use tertiary buttons for navigation and "back" links
  • Ensure buttons have clear, action-oriented labels
  • Use loading states for async actions

‚ùå Don't

  • Don't use multiple primary buttons of different colors in the same section
  • Don't use primary buttons for navigation
  • Don't use vague labels like "Click Here" or "Submit"
  • Don't make buttons too small (minimum 44x44px for touch targets)
  • Don't override button styles with inline CSS

üìê Accessibility

  • All buttons meet WCAG 2.1 AA contrast requirements
  • Buttons have minimum 44x44px touch targets
  • Hover and focus states are clearly visible
  • Icons should have descriptive text (not icon-only buttons)
  • Use disabled state appropriately and provide feedback
